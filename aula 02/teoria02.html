DIV e HTML 5

Na última teoria nós vimos tags textuais e tags de conteúdo. A aula de hoje é sobre tags estruturais. Para entender tags estruturais você precisa ter uma das habilidades mais importantes de um bom dev Front-end. Você precisa saber identificar os componentes de uma página só de bater o olho na página.

Vou te mostrar um exemplo. Eu uso o https://www.trackseries.tv para listar as séries que eu já assisti e as que eu estou assistindo. Vamos ver como é a estrutura do site. Na Pasta dessa teoria eu deixei uma imagem my-tv-series.png abra ela no navegador e veja as minhas marcações.

Vou apresentar alguns elementos. O aside é um elemento caracterizado por ficar no canto da página, não importa se é direita ou esquerda, muitas vezes nem temos um aside. Temos dentro do aside um menu de navegação que no HTML 5 é representado pela tag <nav>, é bem comum ter um menu quando se tem um aside, mas você pode encontrar o menu dentro do header também.

Temos o Header que nesse caso só possui um buscador dentro dele. Super simples.

Temos um container e aqui eu quero que você tenha a consciência que sempre que voce tiver uma lista de alguma coisa, existe um container. Nesse caso temos uma lista de séries, essa lista é o container que "contém" todos os cards.

Por fim, nós temos cada série, é bem comum chamar essa estrutura de card. Então nós dizemos que temos vários cards de série.

Temos uma estrutura chamada de footer que não aparece nesse site, mas a ideia do footer é ficar no final da página, geralmente com informações para contato e links para redes sociais. Olha esse footer do site da Ribon: Ribon desça até o final da página ein!

Claro, eu poderia ir mais fundo no menu de navegação (nav) e dizer que temos uma lista não ordenada dentro dele. Poderia ir mais fundo no card dizendo que cada card tem 3 seções: uma imagem, um título, e uma seção de info dos episódios, embora a ideia aqui seja mostrar as estruturas maiores.

A boa notícia é que as páginas tem estruturas bem parecidas, independente do site, se você pegar o Youtube por exemplo vai ver muitas semelhanças com o trackseries.

E cadê a DIV?

Bom, antes do HTML 5 alguns desses elementos que eu citei não existiam, por exemplo o header não tinha uma tag <header> o menu não tinha uma tag <nav> o aside não tinha tag <aside>. Então como era feito? Nós usávamos a tag <div>. A div é uma tag genérica que pode imitar o comportamento de qualquer outra tag e é provavelmente a tag mais usada no HTML. Isso porque o card de série não tem tag adequeda, não existe uma tag <card-de-série>, então é aqui que a div entra. O HTML tentou com essas tags estruturais dar mais significado ao código de forma que qualquer dev olhe para o código HTML e consiga reconhecer uma estrutura com facilidade, mas claro que estruturas muito específicas não vão ter uma tag, então nós usamos div.

Class e ID

Já pegando o gancho que aparece na foto acima. Em algumas divs tem class e outras tem ID. O ID é usado para identificar um único elemento na página, nós só temos um único header logo o ele é um bom elemento para ter ID. Quando temos vários elementos que se repetem e nós queremos fazer algo com todos esses elementos nós usamos o class. Mas para que eu quero identificar esses elementos? Você vai precisar identificar para colocar CSS nos elementos ou criar algum código javascript, mas isso é o tema das próximas aulas, então por enquanto só saiba que class identifica vários elementos e id identifica um único elemento.
